## Bug Ticket Template

Copy this shape for each new bug:

```md
## BUG-001 - Short bug title

- Status: `Backlog`
- Suggested branch: `fix/short-bug-name`
- Affected area: frontend | backend | shared
- Reported in: page, endpoint, feature, or flow
- Problem:
  - what is happening now
  - what should happen instead
- Reproduction:
  1. step one
  2. step two
  3. observe the failure
- Expected behavior:
  - concise expected result
- Notes:
  - screenshots, logs, constraints, or suspected cause
- Acceptance criteria:
  - clear checks that show the bug is fixed
```

## Rules

- Use bug tickets for defects in existing implemented behavior.
- Use feature entries for new capabilities or scope expansions.
- Move bugs onto `BOARD.md` when they become `Ready`, `In Progress`, or `In Review`.
